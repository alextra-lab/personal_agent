## Issue 3 ‚Äî FRE-344: `display_name` config-driven seeding (~1‚Äì2 hr, Tier-3/Sonnet)

**What:** Add `AGENT_USER_DISPLAY_NAMES_JSON` env var ‚Äî a JSON map of `email ‚Üí display_name` ‚Äî applied idempotently at service startup. Fixes the symptom where non-owner CF Access users get their email local-part as a display name, and likely fixes the session cross-contamination appearance (everyone resolving to the wrong identity).

**Key files:**
- `src/personal_agent/config/settings.py` ‚Äî add new `AppSettings` field
- `.env.example` ‚Äî document the new var
- `src/personal_agent/service/app.py` ‚Äî add bootstrap step after owner bootstrap (~line 466)
- `src/personal_agent/service/user_repo.py` (or wherever `get_or_create_user_by_email` lives) ‚Äî add upsert for display_name
- Neo4j: `memory/service.py` or `memory/identity.py` ‚Äî update `:Person.name` if still email local-part
- `tests/` ‚Äî unit tests for seeding (creates, updates, no-op on subsequent runs)

**New setting to add** (`config/settings.py`):
```python
user_display_names_json: str = Field(
    default="{}",
    description=(
        'JSON map of email ‚Üí display_name for non-owner CF Access users. '
        'Applied idempotently at startup after owner bootstrap. '
        'Example: \'{"alice@x.com":"Alice","bob@x.com":"Bob"}\' '
        "Env var: AGENT_USER_DISPLAY_NAMES_JSON"
    ),
)

@property
def user_display_names(self) -> dict[str, str]:
    import json
    try:
        return json.loads(self.user_display_names_json)
    except (json.JSONDecodeError, ValueError):
        return {}
```

**Bootstrap logic** (add after owner bootstrap block ~line 466 in `app.py`):
```python
# Seed display names for non-owner CF Access users (FRE-344)
for email, display_name in settings.user_display_names.items():
    try:
        async with AsyncSessionLocal() as db:
            uid = await get_or_create_user_by_email(db, email)
        await upsert_display_name(db, uid, display_name)
        # Neo4j: update :Person.name if still the email local-part
        local_part = email.split("@")[0]
        await memory_service.update_person_name_if_default(
            user_id=uid, current_default=local_part, new_name=display_name
        )
    except Exception as e:
        log.warning("display_name_seed_failed", email=email, error=str(e))
```

**Coalesce rule:** Only overwrite `:Person.name` if its current value equals the email local-part (i.e. it was never enriched by entity extraction).
