# FRE-513 — ADR-0088 execution-topology observability spine

**Ticket:** FRE-513 (Approved, Tier-1:Opus, Observability Foundation)
**ADR:** ADR-0088 (Accepted) §D1–D8, Verification
**Builds on:** FRE-452 (route-trace ledger — direct durable write, shipped)
**Strategy (owner decision 2026-06-07):** whole ticket in **one branch**, organized as **ordered commits** matching the phases — not split into sub-tickets. Seam coverage scoped to **primary topology first**; per-sub-agent `(trace_id, task_id)` row fan-out is **deferred** to a single follow-up ticket.

---

## Goal

Replace the per-loop, single-topology live surface with a **mandatory emission seam** + **live projector** so every turn is observable by construction:

1. `observe_topology(...)` — async context manager wrapping `execute_task`; emits `topology_entered` on enter and writes the durable route-trace row + emits `turn_completed` on exit (replacing `_write_route_trace`).
2. **Live projector** — a consumer of `stream:turn.observed` that maintains a per-trace `TurnObservation` and is the *sole* emitter of `turn_status`.
3. `report_degradation(...)` — the single sanctioned "did less" signal; `ExpansionResult.degraded` migrated through it.
4. **Cost cadence (D3)** — live cost driven by `ModelCallCompleted` bus events from the hard-enforced `cost_tracker` boundary; reconciled to `SUM(api_costs)` at `turn_completed`; **FRE-501 per-loop accumulation removed**.
5. **CI teeth (D7)** — a forced fallback produces a durable degradation record + a `turn_status` "degraded" state; a topology running model work outside `observe_topology` fails a test.

Ledger key migrates `UNIQUE(trace_id)` → `UNIQUE NULLS NOT DISTINCT (trace_id, task_id)` (forward-ready; one turn-level row, `task_id=NULL`, for now).

---

## Design decisions (settle ADR open items)

- **`observe_topology` shape:** async context manager (`@asynccontextmanager`) taking the `ExecutionContext` + a resolved `topology` label. Threads alongside `TraceContext` via `ctx`. Chosen over decorator because the durable-row assembly already reads terminal `ctx` state at exit.
- **Topology label source:** derived from `ctx.gateway_output.decomposition.strategy` → `{single→primary, HYBRID→hybrid_fanout, DECOMPOSE→decompose, DELEGATE→delegate}`; `primary` default when gateway absent. Computed once at seam enter, stored on `ctx.topology`.
- **Two sinks (D6):** durable row = direct synchronous Postgres write at the seam (bus-independent, D8). Live = best-effort `bus.publish(STREAM_TURN_OBSERVED, ...)` for `topology_entered` / `model_call_completed` / `degradation` / `turn_completed`.
- **Cost (D3):**
  - Live meter is driven by the **projector** summing `ModelCallCompleted` events (topology-independent — published from `cost_tracker.record_api_call`, the call every model goes through incl. sub-agents). This is the behavioral fix: the meter climbs during expansion.
  - Durable row: `cost_authoritative_usd = SUM(api_costs WHERE trace_id)` (unchanged, correct). `cost_live_usd` = `ctx.turn_cost_usd` for primary turns; for non-primary topologies the local accumulator is not the live source, so the row records `cost_live_usd = cost_authoritative_usd` (the projector, which is the live surface for those turns, reconciles to authoritative). Keeps `cost_reconciled` honest without flooding FRE-514's `not_reconciled` filter.
  - **`not_reconciled` semantics narrowing (codex Q2):** after this change `cost_reconciled=FALSE` detects only a *primary-loop accumulator* drift from authoritative — it no longer detects a projector/bus delivery miss (those rows still read reconciled). This is acceptable because the projector is live-only (D8: a bus miss costs the meter, not durability). Document this in the row docstring + flag a **deferred** projector-health signal (a `projector_events_received` counter) as a follow-up rather than adding a column now.
  - Remove FRE-501 rollups (`ctx.turn_cost_usd += expansion_result.cost_usd` at `executor.py:1821`; `ctx.turn_cost_usd += _sub_agent_cost` autonomous path). Keep ADR-0076's primary per-call `ctx.turn_cost_usd += response.cost_usd`.
- **Degradation severity:** minimal taxonomy now — `severity ∈ {info, warning, critical}`, free-text `reason`, `where` (topology), optional `expected`/`actual`. Full alignment with FRE-451 result-type vocabulary deferred (ADR open decision, joint with FRE-451).
- **Projector gating:** only registered when `event_bus_enabled` and bus is `RedisStreamBus` (mirrors existing consumers). Under `NoOpBus` the durable writes survive; only the live meter is absent (D8).
- **Idempotency:** `turn_status` is a STATE_DELTA (full-state replacement keyed by session) → at-least-once delivery is safe (D4).

## Event model (`stream:turn.observed`)

```python
STREAM_TURN_OBSERVED = "stream:turn.observed"
CG_TURN_PROJECTOR = "cg:turn-projector"

class TopologyEnteredEvent(EventBase):       # trace_id, session_id required
    event_type: Literal["turn.topology_entered"]
    topology: str
    task_id: str | None = None

class ModelCallCompletedEvent(EventBase):     # trace_id, session_id required
    event_type: Literal["turn.model_call_completed"]
    cost_usd: float
    input_tokens: int
    output_tokens: int
    model_role: str | None = None             # purpose/role
    topology: str | None = None

class TurnDegradedEvent(EventBase):           # trace_id, session_id required
    event_type: Literal["turn.degraded"]
    where: str                                 # topology
    reason: str
    severity: Literal["info", "warning", "critical"]
    expected: str | None = None
    actual: str | None = None

class TurnCompletedEvent(EventBase):          # trace_id, session_id required
    event_type: Literal["turn.completed"]
    topology: str
    cost_authoritative_usd: float
    task_id: str | None = None
```

All carry ADR-0074 identity (`trace_id`, `session_id`). `source_component = "observability.topology.seam"` (or `"llm_client.cost_tracker"` for model-call events).

## Verification mapping (ADR-0088)
- Meter climbs during expansion ≈ `SUM(api_costs)` → projector test (Commit 4) + cost emit (Commit 5).
- `turn_status` carries active `topology` → projector test.
- Forced fallback → visible `degraded` + ledger entry → Commit 6 + 7.
- `NoOpBus` → cost + route-trace row still written (direct) → seam test (Commit 3) with NoOp bus.
- No direct `emit_turn_status` from topology loops → Commit 5 (grep-clean) + a test asserting executor no longer imports it.
- Out-of-seam model work fails a test → Commit 7(b).

- **Migration on populated table**: `route_traces` may have rows; `NULLS NOT DISTINCT` add must not fail on existing `trace_id`-unique data (all existing rows have `task_id=NULL`, so `(trace_id, NULL)` stays unique under NULLS NOT DISTINCT — safe). Verify row count before/after on deploy (master).
