# FRE-757 — Rethink the response-rating UX (default "ok", easier selection, error=dark-red / exceptional=gold)

**Ticket:** FRE-757 (Approved → In Progress). Project: Seshat Pedagogical Architecture. Domain: PWA/frontend (build2).
**Backing:** FRE-407 (per-turn 0–3 rating), FRE-426 (server-authoritative hydration). Design pass CLOSED (owner-confirmed 2026-07-03).

## Owner-confirmed design (from ticket comments — treated as fixed, not reopened)

- **Scale:** 3 levels — error / ok / exceptional (not 4).
- **Store mapping (0–3 kept):** error=0, ok=2, exceptional=3. Level 1 ("Low") no longer *offered* by the control but stays a valid stored value for historical rows.
- **ok=2 == today's imputation default** → the FRE-407 flagging metric, ES label mapping, and hydration all stay intact; **no rescale, no backend metric change.**
- **Colors (dark theme, Tailwind tokens):**
  - error: fill `#991b1b`, border `#b91c1c`, ink `#fecaca`, glyph `✕` (bg-red-800 / border-red-700 / text-red-200).
  - ok: fill `#059669`, border `#10b981`, ink `#d1fae5`, glyph `✓` (emerald-600/500/100).
  - exceptional: fill `#d4af37`, border `#e6c34e`, ink `#1a1205`, subtle glow, glyph `★`.
- **Default semantics:** PERSISTED "ok" on every turn (not display-default+imputation). "Every turn carries a real rating written on send." + one-time **backfill** of existing unrated turns to ok=2.
- **Sizing:** visible icon core ~24px inside a transparent 44×44px hit target, ~4px gaps (Apple HIG 44pt / WCAG 2.5.5).
- **Behavior (unchanged from FRE-407/426):** persistently visible in the assistant footer (never hover-gated), optimistic paint + pulse on confirm, re-rating overwrites, prior ratings hydrate from history, never renders mid-stream. Label via hover/long-press + aria-label.

## Mechanism decisions (REVISED per codex plan-review 2026-07-03)

**M-A — "persist ok on send" → LIVE-DONE-only auto-persist + BACKEND create-if-absent write.**
*(Codex rejected the original mount-effect: history hydration marks every assistant turn `complete:true`, so a mount-based auto-POST would burst across all unrated historical turns, and a late default-`2` could **clobber an explicit manual `0`/`3`** — writes overwrite by `doc_id=trace_id`, and same-value suppression is event-only, not write-only.)*
- **Frontend fires the default write ONLY from the live `DONE` transition** (`useSSEStream.ts` DONE case), keyed by the just-completed trace/session, with a per-trace client guard so it fires at most once. It is NOT a generic `TurnRating` mount effect → historical/replayed turns never auto-post.
- **Backend gains a create-only default mode.** `submitTurnRating` sends `default: true` for the auto-write; the endpoint uses ES `op_type=create` (doc_id=trace_id): if a rating doc already exists it 409s → treated as success-noop. This is atomic at the ES layer (no refresh-lag race): a manual rating and the default can arrive in any order and the explicit rating always wins (manual uses the existing overwrite/`index` path; default can only create-if-absent). No bus event on default writes.
- This makes deploying the UI **safe regardless of backfill ordering** — no bursts, no clobber — so backfill timing is decoupled (see Scope).

**M-B — backfill field-fidelity (correctness, not preference).** Backfilled docs MUST copy from the source `model_call_completed` event: **`rated_at` = the event's original `@timestamp`** (NOT `now` — else all history collapses into the current metric window), **`prompt_callsite`** (the aggregation reads the rating doc's *own* callsite with `missing:"unknown"` and the engine *excludes* `unknown` — a callsite-less backfill doc lands in a junk excluded bucket), `session_id`, and the `prompt_*` hashes/component_ids for other consumers. `doc_id=trace_id`; use `op_type=create` to skip already-rated trace_ids (idempotent, and consistent with M-A's default write).

**M-C — hydration of legacy stored value 1 ("Low", historical only).** *(Codex: mapping `1`→muted-ok hides a below-default negative signal.)* Leave the stored value **unchanged**; hydrate `1` as a distinct **legacy-low** visual nearer *error* (muted/outlined error-side chip, aria-label "legacy low rating"); the next click overwrites it with `0/2/3`. Do NOT migrate `1`→`0` (overstates) or `1`→`2` (erases the negative signal).

**M-D — replay-gap hydration must preserve `rating` (codex-found bug).** `useSSEStream.ts` REPLAY_GAP mapping currently drops `rating` (only sets `complete:true`), so a reconnect would strip prior ratings. Add `rating: m.rating` there (mirrors the normal history hydration) + a regression test. Without this, a replayed turn would look unrated (though M-A's live-only trigger means it would NOT auto-post — the harm is a wrong faint UI, not a clobber).

## Scope / PR cut

Codex flagged the original "deploy PWA then backfill" ordering as the risk. The **revised M-A design removes that coupling**: auto-persist is live-`DONE`-only (never fires on historical/replayed mounts) and the default write is create-only (can't clobber). So the UI is **safe to deploy before or after the backfill**. → **Keep as ONE PR** (frontend control + live default persist + backend create-only mode + backfill script), which is cohesive and safe. *(Split option, if the owner prefers: PR-1 backend create-only + backfill, PR-2 UI — not required for safety now.)*
